## 🔍 О проекте

Этот проект реализует нейросетевой автокомплит для shell-команд на основе реальной истории команд ZSH. Модель обучается предсказывать следующий **токен** (часть команды) по уже введённому контексту — аналогично тому, как работает автодополнение в IDE или поисковой строке.

**Примеры предсказаний:**

```
Варианты продолжения «sudo systemctl»:
  1. status  (уверенность: 43.18%)
  2. stop  (уверенность: 13.75%)
  3. restart  (уверенность: 12.12%)
──────────────────────────────
```

## ⚙️ Как это работает

### 1. Предобработка данных (`scripts/pre_clean.py`)

ZSH сохраняет историю в специальном расширенном формате:
```
: <timestamp>:<duration>;<command>
```
Скрипт `pre_clean.py` очищает этот формат — убирает метаданные и оставляет только **сами команды** с двумя и более словами. Результат сохраняется в `dataset/history_pre_clean`.

### 2. Токенизация

Каждая команда разбивается на токены с помощью `RegexpTokenizer(r"\w+|[^\w\s]")` — регулярное выражение выделяет слова и отдельные специальные символы (`/`, `-`, `|`, `.` и т.д.).

Пример токенизации:
```
"git push origin main"  →  ["git", "push", "origin", "main"]
"ls -la /etc"           →  ["ls", "-", "la", "/", "etc"]
```

### 3. Построение словаря

Из всех токенов строится словарь `vocab: dict[str, int]`, включающий специальные токены:
- `<PAD>` (id=0) — паддинг последовательностей
- `<UNK>` (id=1) — неизвестный токен

### 4. Формирование обучающих примеров

Для каждой команды генерируются все возможные префиксы:
```
["git", "push", "origin", "main"]
→ (["git"],           target: "push")
→ (["git", "push"],   target: "origin")
→ (["git", "push", "origin"], target: "main")
```
Итоговый датасет: **11 256 примеров**, длина контекста — **61 токен**.

### 5. Обучение и предсказание

Модель принимает на вход закодированный (и паддированный) контекст и предсказывает распределение вероятностей над всем словарём. Функция `predict_next()` возвращает `top_k` наиболее вероятных следующих токенов.

## 🏗️ Архитектура модели

```
Input (padded token sequence)
        ↓
  Embedding Layer (vocab_size → 64)
        ↓
  LSTM Layer 1  (64 → 128, dropout=0.2)
        ↓
  LSTM Layer 2  (128 → 128, dropout=0.2)
        ↓
  Fully Connected (128 → vocab_size)
        ↓
  Softmax → Top-K predictions
```

| Параметр         | Значение         |
|------------------|-----------------|
| Embedding dim    | 64              |
| Hidden dim       | 128             |
| LSTM layers      | 2               |
| Dropout          | 0.2             |
| Optimizer        | Adam (lr=1e-3)  |
| Loss             | CrossEntropyLoss (label_smoothing=0.1) |
| LR Scheduler     | ReduceLROnPlateau (factor=0.5, patience=5) |
| Grad clipping    | max_norm=1.0    |

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone <repo-url>
cd zsh_history_rnn
cp ~/.zsh_history ./dataset/zsh_history

python -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. Предобработка истории ZSH

Поместите вашу `~/.zsh_history` в папку `dataset/` и запустите:

```bash
python scripts/pre_clean.py
```

### 3. Обучение модели

Запустите ноутбук `main.ipynb` в Jupyter:

```bash
jupyter notebook main.ipynb
```

Или выполните ячейки последовательно для:
- загрузки и токенизации данных
- обучения LSTM-модели
- оценки качества
- интерактивного предсказания

### 4. Инференс (предсказание)

Для использования уже обученной модели (веса в `metadata/`):

```bash
python main.py
```

```
Введите начало команды: docker
Варианты продолжения «docker»:
  1. compose  (уверенность: 24.57%)
  2. run      (уверенность: 18.33%)
  3. ps       (уверенность: 12.10%)
──────────────────────────────
```

## 🔧 Параметры обучения

Параметры находятся в начале `main.ipynb` и `main.py`:

```python
EMBEDDING_DIM  = 64      # Размерность эмбеддингов
HIDDEN_DIM     = 128     # Размер скрытого состояния LSTM
BATCH_SIZE     = 32      # Размер батча
EPOCHS         = 50      # Количество эпох
LR             = 1e-3    # Скорость обучения
TEST_SIZE      = 0.2     # Доля тестовой выборки
RANDOM_STATE   = 42      # Seed для воспроизводимости
```
